clsFileStream


    This class gives you easy access to sequential file operations such as opening a file for reading or writing and processing it libe by line as opposed to reading the whole file in in one shot. These techniques can be very useful (if not mandatory) for large files that would take up to much memory to read in all at once.

    This library offers functionality similar to that of the windows script host Scripting.FileSystemObject. There are however some important differences.

  1. It does not have a long nested layout and is easier to use
  2. It supports binary functions on files
  3. You have to be careful when early binding to the WSH libraries because they are dependant on what version of IE and Windows you have installed and could be an extra dependancy you have to install on the users computer. Also, some versions of the WSH are not yet ported to XP!

Property Get FileHandle() As Long

Returns the open file handle

Property Get EndOfFile() As Boolean

Returns true when End of File is reached.

Property Get LengthOfFile() As Long

Returns length of opened file

Sub fClose()

Closes the file when you are done with it. (or automatically closes when program exits)

Sub fOpen(path, t As openType)

Opens a file for read,write, append or binary

Function ReadLine(Optional lineCount = 0) As String

Reads x Lines from the file. Called with no arguments, it reads on line at a time.

Sub WriteLine(str)

Writes a single line to the file. (if your string contains carriage returns can be > 1 line)

Sub WriteExpandedString(ByVal str)

Writes to the file and expands \n -> vbcrlf and \t -> vbtab. Useful for writing C style strings.

Sub WriteDivider(Optional divChar = "-", Optional num = 75)

Usefull for report writing. Writes a divider to the page using the character and number specified.

Sub WriteBlankLine(Optional lineCount = 0)

Writes blank lines to the document.

Sub BinWriteDump(it, Optional offset = "")

This will dump the blob you give it to file. If "it" is a userdefined type then it will be an object dump (see the article on my site). Will not dump arrays. If offset ="" then it will dump it at teh current file offset, otherwise itwill place it at the offset specified.

Sub BinGetByteArray(buffer() As Byte, Optional offset = "")

Fills the byte array you pass in with the contents of the file. offset variable as defined above.

Sub BinGetStrArray(buffer() As String, Optional offset = "")

Returns a string array of the characters in the file. you set buffer size based on amount of characters to read. offset as defined above.

Function BinGetChar() As Byte

Returns the next byte to be read in the file.

Sub BinWriteNChar(b As Byte, Optional num As Long = 0)

Returns the specified byte value to file n times.
