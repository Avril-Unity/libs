Welcome to the VB developers Kit.


    This Library is an ActiveX Dll that can be utilized from any Language that supports ActiveX objects such as Visual Basic, Delphi, C++, Even Interpreted Languages such as VBS and JS in the Windows Script Host Enviroment !

    This Library extends and simplifies a host of programming tasks and makes even the newest developer comfortable and productive.

    The Simple function names and intutive interfaces make coding with the VB Dev Kit a breeze. This is because I have developed these libraries over the coarse of the last 4 years, and I have developed then to be exactly what I needed.

The VB Developers kit includes the following classes:

clsFileSystem - Manipulate and access the file system with ease. Full Intellisense fully debugged library, makes reading and writing files as painless as can be. And without the version conflicts associated with the Scripting.FileSystemObject !
 
clsFileStream - Sequentially read through files line by line or byte by byte. Also has functions to Write line out, great for report generation!
 
clsCmnDlg - Why Distribute an external ocx for the common dialog functions? Wraps the Windows API for an easy to use built in addition to the library.
 
clsCmdLine - Easily and efficiently gain access to the programs command line.
 
clsRegistry - Gain full access and harness the power of the registry.
 
clsStrings - This is a large string parsing class that is guaranteed to clean up and simply your string parsing routines ! Makes complex parsings an absolute joy.
 
clsIniFile - This trumps the power of the Windows API in terms of working with ini files. Gives you the power to enumerate sections and enumerate keys.


    The VB Developers Kit is written in pure VB and Windows Api which means that it needs no other components (other than the VB 6 Runtimes of course) to operate. The VB runtimes come installed by default on Windows ME,XP, and 2000 machines. For Older machines, such as Win95, 98 and NT they will have to have the VB 6 Runtimes installed before you use the VB Developers Kit.
